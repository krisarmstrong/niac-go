## Part 1: Command Structure Redesign (Week 1)

### Current State
```bash
./niac [interface] [config]
./niac --interactive [interface] [config]
```

### New Command Structure

```bash
# Main commands
niac run [config]           # Start simulation
niac validate [config]      # Validate configuration
niac convert [file]         # Convert legacy .cfg to YAML
niac menu                   # Launch interactive menu (NEW)
niac wizard                 # Guided setup wizard (NEW)
niac templates              # List/manage templates (NEW)
niac devices                # Device management (NEW)

# Simulation control
niac run --interface eth0 --config my-network.yaml
niac run --interactive      # Launch with TUI
niac run --daemon           # Background mode
niac run --watch            # Auto-reload on config change

# Configuration management
niac config validate my-network.yaml
niac config create --template basic
niac config create --wizard
niac config show my-network.yaml
niac config edit my-network.yaml

# Device operations
niac devices list           # List all defined devices
niac devices show router-1  # Show device details
niac devices test router-1  # Test single device
niac devices export         # Export device inventory

# Template system
niac templates list         # Show available templates
niac templates show basic   # Display template
niac templates create       # Create custom template

# Utility commands
niac interfaces             # List available network interfaces
niac version --verbose      # Detailed version info
niac health                 # Self-diagnostics
niac doctor                 # Environment check
```

### Implementation
- Use cobra for command structure
- Add flags with sensible defaults
- Consistent command patterns (verb-noun)
- Rich help text with examples
- Shell completion (bash, zsh, fish)

## Part 7: Configuration Management (Week 5)

### Config Commands

```bash
# Show config in formatted view
niac config show my-network.yaml

# Edit config with $EDITOR
niac config edit my-network.yaml

# Convert legacy .cfg to YAML
niac config convert old-config.cfg --output new-config.yaml

# Merge multiple configs
niac config merge base.yaml extra-devices.yaml --output combined.yaml

# Extract devices
niac config extract my-network.yaml --device router-1 --output router-1.yaml

# Compare configs
niac config diff old-config.yaml new-config.yaml

# Lint/format config
niac config format messy-config.yaml
```

### Config Version Control

Track configuration changes:
```bash
# Save config snapshot
niac config snapshot my-network.yaml

# List snapshots
niac config snapshots list

# Restore snapshot
niac config snapshots restore my-network.yaml 2025-01-11-123456

# Show diff
niac config snapshots diff my-network.yaml 2025-01-11-123456
```
