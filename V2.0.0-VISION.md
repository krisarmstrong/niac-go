## Vision Statement

NIAC-Go v2.0 will be a **complete network simulation platform** with:
- **Modern Web UI** - Full-featured web interface for configuration and monitoring
- **REST API** - Programmatic access to all functionality
- **Real-time Monitoring** - Live dashboards with WebSocket updates
- **Multi-user Support** - Team collaboration with role-based access
- **Enterprise Features** - SSO, audit logging, compliance
- **Cloud Native** - Docker, Kubernetes, scalable architecture

## Part 1: Web Interface (Weeks 1-4)

### Technology Stack

**Frontend**:
- React 18 or Vue 3 (TBD based on team preference)
- TypeScript for type safety
- Tailwind CSS for styling
- Chart.js / D3.js for visualizations
- React-Flow / Vue-Flow for topology diagrams
- WebSocket client for real-time updates

**Backend**:
- Go with Gin or Echo framework
- RESTful API with OpenAPI/Swagger docs
- WebSocket server (Gorilla WebSocket)
- JWT authentication
- CORS middleware

## Part 4: Database & Persistence (Weeks 5-6)

### Database Schema

**PostgreSQL** for structured data:

```sql
-- Users
CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    username VARCHAR(255) UNIQUE NOT NULL,
    email VARCHAR(255) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    role VARCHAR(50) NOT NULL,  -- admin, user, viewer
    created_at TIMESTAMP DEFAULT NOW(),
    last_login TIMESTAMP
);

-- Configurations
CREATE TABLE configurations (
    id SERIAL PRIMARY KEY,
    name VARCHAR(255) NOT NULL,
    description TEXT,
    yaml_content TEXT NOT NULL,
    created_by INTEGER REFERENCES users(id),
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

-- Simulations
CREATE TABLE simulations (
    id SERIAL PRIMARY KEY,
    name VARCHAR(255) NOT NULL,
    config_id INTEGER REFERENCES configurations(id),
    interface VARCHAR(50),
    status VARCHAR(50),  -- running, stopped, error
    started_at TIMESTAMP,
    stopped_at TIMESTAMP,
    started_by INTEGER REFERENCES users(id)
);

-- Devices
CREATE TABLE devices (
    id SERIAL PRIMARY KEY,
    simulation_id INTEGER REFERENCES simulations(id),
    name VARCHAR(255) NOT NULL,
    type VARCHAR(100),
    ip_address VARCHAR(45),
    mac_address VARCHAR(17),
    status VARCHAR(50)
);

-- Metrics (time-series data)
CREATE TABLE metrics (
    id SERIAL PRIMARY KEY,
    simulation_id INTEGER REFERENCES simulations(id),
    device_id INTEGER REFERENCES devices(id),
    timestamp TIMESTAMP NOT NULL,
    metric_type VARCHAR(100),
    value JSONB
);

-- Audit Logs
CREATE TABLE audit_logs (
    id SERIAL PRIMARY KEY,
    user_id INTEGER REFERENCES users(id),
    action VARCHAR(100),
    resource_type VARCHAR(100),
    resource_id INTEGER,
    details JSONB,
    ip_address VARCHAR(45),
    timestamp TIMESTAMP DEFAULT NOW()
);

-- Templates
CREATE TABLE templates (
    id SERIAL PRIMARY KEY,
    name VARCHAR(255) NOT NULL,
    description TEXT,
    yaml_content TEXT NOT NULL,
    category VARCHAR(100),
    devices_count INTEGER,
    created_by INTEGER REFERENCES users(id),
    created_at TIMESTAMP DEFAULT NOW(),
    is_public BOOLEAN DEFAULT FALSE
);
```

### Data Retention

- Real-time metrics: 24 hours (in-memory + DB)
- Historical metrics: 30 days (DB)
- Audit logs: 1 year
- Configurations: Unlimited
- Simulations: 90 days (configurable)

```dockerfile
FROM golang:1.21-alpine AS builder
```
